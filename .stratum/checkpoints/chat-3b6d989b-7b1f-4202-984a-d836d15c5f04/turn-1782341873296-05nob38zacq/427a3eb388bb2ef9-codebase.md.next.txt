## Purpose
Quantum Computing + Mechanics Research Archive — a curated, reference-style website modeled after research.surajgaud.com/overview. Built with Next.js 16, TypeScript, and Tailwind CSS v4.

## Architecture
- **Framework**: Next.js 16 (App Router) with TypeScript
- **Styling**: Tailwind CSS v4 (CSS-first configuration, no tailwind.config.js)
- **Content**: TypeScript data modules in `src/data/`
- **Components**: Reusable UI components in `src/components/`
- **Layout**: Fixed sidebar (w-56, slate-900) + fixed header (h-14) + scrollable main content area
- **Deployment**: Static export-ready (output: 'export' can be added in next.config.ts)
- **Icons**: lucide-react

## Important Files and Directories

### Config Files
- `next.config.ts` — Next.js configuration (turbopack root set)
- `tsconfig.json` — TypeScript with `@/*` path alias mapping to `./src/*`
- `postcss.config.mjs` — PostCSS with @tailwindcss/postcss
- `eslint.config.mjs` — ESLint flat config
- `package.json` — Dependencies: next 16.2.9, react 19, tailwindcss 4, lucide-react

### src/app/ (App Router Pages) — All Implemented with Real Data
- `layout.tsx` — Root layout with Sidebar, Header, and main content slot
- `page.tsx` — Home page (redirects to /overview)
- `globals.css` — Tailwind v4 CSS with custom design system variables + tag/role utility classes
- `overview/page.tsx` — **Server component**: Welcome section, stats bar (4 stat cards), collapsible CurriculumSection with 9 modules
- `resources/page.tsx` — **Client component**: FilterBar by category (Book, Course, Video Playlist, Platform), grid of ResourceCards, result count
- `community/page.tsx` — **Client component**: FilterBar by role (Researchers, Educators, Builders, Institutes), grid of CommunityCards
- `articles/page.tsx` — **Client component**: FilterBar by tag, grid of ArticleCards
- `roadmap/page.tsx` — **Server component**: Vertical timeline with numbered circles, module cards with lesson lists and type icons
- `tools/page.tsx` — **Client component**: FilterBar by category (SDK, FRAMEWORK, LANGUAGE, SIMULATOR), grid of ToolCards
- `papers/page.tsx` — **Server component**: Curated list of 12 foundational papers as styled linked cards with year and PAPER tag
- `glossary/page.tsx` — **Client component**: FilterBar by category (Fundamentals, Formalism, Gates & Circuits, Algorithms, etc.), 3-column grid of GlossaryTerms
- `newsletters/page.tsx` — **Client component**: FilterBar by tag, grid of NewsletterCards

### src/components/ (Shared UI Components)

- `Header.tsx` — Fixed top header with brand, search button (⌘K), GitHub star button
- `TagBadge.tsx` — Inline span with tag-{tag} CSS class for colored badges
- `FilterBar.tsx` — Horizontal scrollable row of filter buttons with All + categories
- `ResourceCard.tsx` — White card for resources with title, author, description, category badge, tag badges
- `CommunityCard.tsx` — Card with avatar initials, name, role badge, description, tags
- `ArticleCard.tsx` — Card for articles with title, author, description, tags
- `ToolCard.tsx` — Card for tools with title, category badge, description, tags
- `GlossaryTerm.tsx` — Card for glossary terms with term name, category badge, definition
- `NewsletterCard.tsx` — Card for newsletters with name, frequency badge, description, tags
- `CurriculumSection.tsx` — Module cards for overview page with collapsible lesson lists and type icons

- `resources.ts` — exports `resources: Resource[]` (25 items: 9 books, 7 courses, 6 videos, 5 platforms)
- `curriculum.ts` — exports `curriculum: Module[]` (9 modules, 55 lessons)
- `community.ts` — exports `communityMembers: CommunityMember[]` (15 members with RES/EDU/BUILD/LEAD roles)
- `articles.ts` — exports `articles: Article[]` (12 articles)
- `tools.ts` — exports `tools: Tool[]` (10 tools: SDK/FRAMEWORK/LANGUAGE/SIMULATOR)
- `glossary.ts` — exports `glossaryTerms: GlossaryTerm[]` (30 terms in 7 categories)
- `newsletters.ts` — exports `newsletters: Newsletter[]` (6 newsletters)

## Build, Test, and Run Commands
- `npm run dev` — Start development server
- `npm run build` — Production build (verified passing with 0 errors, all 13 pages static)
- `npm run start` — Start production server
- `npm run lint` — Run ESLint

## Data Flow and Integrations
- All data is statically defined in TypeScript files under `src/data/`
- Pages import data directly from their respective data modules
- Static generation at build time (all pages are '○ Static')
- Client pages (`"use client"`): resources, community, articles, tools, glossary, newsletters — use useState/useMemo for FilterBar filtering
- Server pages: overview (uses client CurriculumSection child), roadmap (timeline), papers (curated list)

## Conventions and Patterns
- TypeScript strict mode enabled
- Path alias `@/*` maps to `./src/*`
- Tag system: uppercase strings (VIDEO, PAPER, TUT, GUIDE, NOTES, BOOK, COURSE, PLATFORM, etc.)
- Roles: RES (Researcher), EDU (Educator), BUILD (Builder), LEAD (Leader/Institute)
- Tool categories: SDK, FRAMEWORK, LANGUAGE, SIMULATOR
- Consistent page structure: space-y-8 wrapper, h1 heading, subtitle, FilterBar section, content grid with result count
- Grid patterns: 2-column (md:grid-cols-2) for most pages, 3-column (lg:grid-cols-3) for glossary
- Empty state handling: centered message when filter returns 0 results
- External links use target="_blank" and rel="noopener noreferrer"
